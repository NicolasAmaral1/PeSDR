# FE-01b Cutover Playbook — flipping a tenant v1 -> v2

> Audience: operator on call. Pre-req: FE-01b merged + deployed.

## What this is

FlowEngine v2 replaces the LangGraph runtime for v2-flagged tenants.
Tenants with `architecture_version=1` continue on the legacy path until
FE-02 removes it.

## Status of the cutover

**FE-01b foundation + FE-01b cutover wiring are both in place.**

What works:
- `run_turn` orchestrator wired end-to-end (Tasks 1-17)
- Smoke E2E test drives 3 happy turns against fake adapter (Task 22)
- `process_lead_inbox` routes by `architecture_version`; the v2 branch
  fully wires TreeFlow + version + LLM (with secrets via SopsLoader) +
  adapter + GuardrailConfig + opt_out_keywords and calls `run_turn`.
- Simulate CLI `--arch-v2` plugs into Typer and dispatches an inline REPL
  (`_run_v2`) that calls `simulate_v2_turn` per inbound text.
- Avelum tenant + TreeflowVersion seedable via
  `uv run python scripts/seed_avelum_v2.py` (idempotent).

What still depends on operator config (outside code):
- `tenants/avelum/secrets.enc.yaml` with `openai_key` (matching
  `tenant.yaml`'s `api_key_ref: secrets/openai_key`) — required for the
  v2 branch + the simulate REPL to actually call OpenAI.

## Manual flip on a tenant

The seed script sets `architecture_version=2` for Avelum on first run.
For tenants already in the DB at v1, run:

```sql
UPDATE tenants SET architecture_version = 2 WHERE slug = '<slug>';
```

## Pilot smoke (Avelum)

Pre-req: `sops` CLI + age key on the box, `tenants/avelum/secrets.enc.yaml`
populated with `openai_key`.

```bash
uv run python scripts/seed_avelum_v2.py        # one-time idempotent seed
uv run ai-sdr simulate \
  --tenant avelum --treeflow avelum_sdr \
  --lead pilot-1 --arch-v2
# type a message, /quit to exit
```

## Monitoring playbook (first hour after flip)

1. `outbound_messages` per tenant per minute — expect steady rate matching inbound.
2. `talks.status = 'requires_review'` count — near-zero unless guardrails recur.
3. Logs grepped for `turn_escalated_via_guardrails` — each entry indicates a CorrectionEscalation.
4. Spot-check `outbound_messages.body_text` — confirm persona matches.

## Rollback

```sql
UPDATE tenants SET architecture_version = 1 WHERE slug = 'avelum';
```

Roll back ONLY if v2 is clearly broken. Otherwise prefer to fix forward.

## Known limitations of FE-01b

- No objection treatment runtime (FE-03)
- No Sentinel runtime (FE-04)
- No voice (FE-05)
- No event emission (FE-06)
- No HITL response_reviews runtime (FE-07)

Their slots are reserved in FE-01a schema; turning them on does not
require migrations.
