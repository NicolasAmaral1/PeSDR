# FE-01b Cutover Playbook ‚Äî flipping a tenant v1 -> v2

> Audience: operator on call. Pre-req: FE-01b merged + deployed.

## What this is

FlowEngine v2 replaces the LangGraph runtime for v2-flagged tenants.
Tenants with `architecture_version=1` continue on the legacy path until
FE-02 removes it.

## Status of the cutover

**FE-01b is the foundation. The actual cutover wiring is incomplete.**

What works in FE-01b:
- `run_turn` orchestrator wired end-to-end (Tasks 1-17)
- Smoke E2E test drives 3 happy turns against fake adapter (Task 22)
- `process_lead_inbox` BRANCHES on `architecture_version` (Task 18) but
  the v2 branch is a STUB that logs and returns ‚Äî full dep wiring (TreeFlow
  + version + LLM + adapter + guardrails + opt_out_keywords) was not done
  in FE-01b. See Known Issues below.
- Simulate CLI `--arch-v2` via `simulate_v2_turn` helper (Task 21).

What is NOT yet operational:
- A real v2 inbound from WhatsApp would route to `_run_v2_pipeline` but
  hit the stubbed body and skip processing.
- Resolving the right TreeflowVersion for a tenant is not wired.
- `tenant.llm.default` needs to be loaded via TenantLoader and passed in.
- `guardrails` block in tenant.yaml needs to be parsed ‚Üí GuardrailConfig.

## Known issue: legacy step shim

Task 18 added a `_run_legacy_pipeline` indirection that lazily imports
`ai_sdr.treeflow.runtime.step`. The actual module exposes
`TalkFlowRuntime.step()` (a class method), not a module-level function.
The tests mock both helpers so this doesn't break CI, but a real v1
inbound on a v1 tenant would hit the broken import.

Fix when wiring real cutover: rewrite `_run_legacy_pipeline` to
instantiate `TalkFlowRuntime` and call its `.step(...)` method.

## Monitoring playbook (first hour after flip)

1. `outbound_messages` per tenant per minute ‚Äî expect steady rate matching inbound.
2. `talks.status = 'requires_review'` count ‚Äî near-zero unless guardrails recur.
3. Logs grepped for `turn_escalated_via_guardrails` ‚Äî each entry indicates a CorrectionEscalation.
4. Spot-check `outbound_messages.body_text` ‚Äî confirm persona matches.

## Rollback

```sql
UPDATE tenants SET architecture_version = 1 WHERE slug = 'avelum';
```

Roll back ONLY if v2 is clearly broken. Otherwise prefer to fix forward.

## Known limitations of FE-01b

- No objection treatment runtime (FE-03)
- No Sentinel runtime (FE-04)
- No voice (FE-05)
- No event emission (FE-06)
- No HITL response_reviews runtime (FE-07)

Their slots are reserved in FE-01a schema; turning them on does not
require migrations.
